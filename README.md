# Emoji Mosaic Bot

Бот режет картинку на квадратики, делает из них набор кастомных эмодзи
в Telegram и сразу присылает собранную мозаику обратно в чат.

## Как это работает

1. Пользователь шлёт боту фото.
2. Бот спрашивает размер сетки (например `5x5`).
3. Картинка приводится к квадрату (с прозрачными полями, без обрезки)
   и режется на N×N равных квадратов 100×100px.
4. Бот создаёт `custom emoji sticker set` через Bot API
   (`createNewStickerSet` с `sticker_type=custom_emoji`), владелец сета —
   сам пользователь (его `user_id`).
5. Бот отправляет сообщение, где кусочки расставлены построчно —
   они визуально складываются в исходную картинку.

## ⚠️ Важные ограничения Telegram, которые нельзя обойти

- **Custom emoji полноценно видны только у пользователей с Telegram
  Premium.** У всех остальных вместо мозаики будет серый прямоугольник-
  плейсхолдер. Это ограничение самого Telegram, не бота.
- Размер каждого эмодзи строго 100×100px.
- Максимум 200 эмодзи в одном наборе (поэтому сетка ограничена до 12×12=144
  в коде, с запасом).
- Имя набора (`short_name`) должно быть уникальным во всём Telegram и
  заканчиваться на `_by_<username бота>` — это уже сделано в коде
  автоматически через uuid.
- Боту нужны права на работу со стикерами — выдаются автоматически,
  отдельно включать ничего не нужно.

## Установка и запуск

### 1. Создать бота

В Telegram напиши [@BotFather](https://t.me/BotFather):
```
/newbot
```
Следуй инструкциям, получишь токен вида `123456:ABC-DEF...`.

### 2. Установить зависимости

```bash
pip install python-telegram-bot==21.* Pillow --break-system-packages
```

### 3. Запустить локально (для теста)

```bash
export BOT_TOKEN="твой_токен_от_BotFather"
python3 bot.py
```

Бот будет работать пока запущен процесс (polling-режим). Для теста этого
достаточно — просто напиши боту в Telegram.

### 4. Задеплоить, чтобы бот работал всегда (24/7)

Локальный запуск перестанет работать, если закрыть терминал/ноутбук.
Варианты бесплатного/простого хостинга:

**Railway.app** (проще всего):
1. Залей код в GitHub-репозиторий (файл `bot.py` + `requirements.txt`).
2. На [railway.app](https://railway.app) → New Project → Deploy from GitHub.
3. В Variables добавь `BOT_TOKEN` = твой токен.
4. Railway сам поставит зависимости из `requirements.txt` и запустит
   `python3 bot.py`.

**Render.com** — аналогично, через Background Worker (не Web Service,
так как боту не нужен открытый порт).

**Свой VPS** — через `systemd`-сервис или `screen`/`tmux`, чтобы процесс
не завершался при отключении SSH.

### requirements.txt

```
python-telegram-bot==21.6
Pillow==10.4.0
```

## Возможные доработки

- Поддержка прямоугольных сеток (не только N×N) — сейчас квадрат для
  простоты и предсказуемости результата.
- Лимит на размер входящей картинки (защита от слишком больших файлов).
- Хранение `pending_images` в базе вместо памяти процесса — нужно, если
  бот будет рестартоваться часто или работать на нескольких воркерах.
- Команда `/grid 5x5`, чтобы не ждать отдельного текстового сообщения.
